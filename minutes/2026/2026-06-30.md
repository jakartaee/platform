* Debate about how many implementations are needed  
  * Had this discussion with REST, CDI, NoSQL and Persistence  
  * There are no requirement to have more than one implementation  
  * Over time vendors may coalesce to one implementation  
  * Starting a second / third implementation is time consuming especially as a specification grows  
  * If there is only one implementation, others are welcome to contribute to it.  
  * Reasons to have a new implementation include licensing and governance  
  * Need to have multiple vendors contributing to an implementation if there is only one implementation to get diverse thought from the spec experts
