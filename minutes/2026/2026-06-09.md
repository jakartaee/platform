## Config

* Jan presents the “future directions presentation” on the topic of MicroProfile Config/Jakarta Config   
  * MP Config current version depends on Jakarta EE 9 right now which doesn’t have Core Profile which really messes with dependencies  
  * Moving MP or MP Config to Jakarta is something that can be started now, but likely wouldn’t be part of the EE 12 platform, but would be in preparation for EE 13  
  * Config Core and Config CDI split is important in order for CDI to be able to use Config and not cause a circular dependency.  As such Config cannot stay in MP for CDI to use it  
  * Having a separate Jakarta Config spec and MP Config (option not presented by Jan) would be the worst scenario for everyone
